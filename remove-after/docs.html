Get started

Zentla is a provider-agnostic billing layer. You keep your domain model stable; adapters handle Stripe/Zuora differences. This guide gets you to “usage recorded + entitlements enforced” fast.

TypeScript Node.js Idempotency-first
Principle: money-safe semantics

Treat every billing-side action as potentially retried. Always send an idempotency key. Zentla is designed so replaying events is safe.

Install

npm i zentla

Initialize the client

import { Zentla } from "zentla";

const z = new Zentla({
  apiKey: process.env.ZENTLA_API_KEY,
});

Record usage (replay-safe)

Record a usage event with an idempotency key so the exact same event can be retried safely. 

await z.usage.record({
  tenantId: "tnt_123",
  metric: "api_calls",
  quantity: 1200,
  idempotencyKey: "evt_0001",
});

Enforce entitlements in-app

Entitlements let you gate features consistently, regardless of provider. 

const ent = await z.entitlements.get("tnt_123");

if (!ent.features.advancedReports) {
  throw new Error("Upgrade required");
}
